# Project Context

This directory is for local project context used while working with Nimbly and AI coding agents: requirements notes, design references, research, meeting notes, imported documents, and other background material that helps explain the project.

By default, files in this directory are local only and are not committed to the core repository. The only tracked files here are this readme and the ignore rules that keep the directory available without publishing private context.

Use this root `.context` directory for context that belongs to a local checkout, operator, client, environment, or private workflow.

For context that should be deliberately shared with everyone working on the application, use `ext/.context` in the nested ext repository instead. Files in `ext/.context` belong to the application repo and should only contain project context that is safe and useful to share with that team.
